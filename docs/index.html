About this wee project...

This little grid system was born out of conflict between those that want double padding on their grid using a very well known framework and those that wanted a little more customization.

The customization was important: Why constrain the grid to $X pixels wide? Why can we not choose double padded or single padded columns for more layout options? Why is the milk in the fridge off, again?

I took this on as my own wee one nighter project to:

- Delve into the world of the gird instead of just accepting the status quo
- Learn a bit more about sass and how to incorporate it into my workflow
- To avoid wasting another night spent playing Stardew Valley

The Sassy Grid

By default this grid is a fluid full width grid. Once it hits below 480px (large mobile), it stacks all columns as full width. There is no mobile columns by default - you'll have to add your own as needed.

Default Breakpoints

These are the default breakpoints. If you want full fluid from mobile up, just use the col--sm-size class. If you're comfortable with sass, you can open up the sassy-grid.scss file and alter the variables at the top to suit your needs, heck, @import and @extend if that's what you're into...

Col class Breakpoint
xs (mobile) < 480px
sm (tablet) 480px - 991px
md (large tab, small laptop) 992px - 1023px
lg (desktop and beyond) > 1024px
